## 2024-05-18 - Static Derived State Calculation
**Learning:** This codebase heavily relies on importing large static objects (e.g. `SUBSTANCES` in `src/data/substances.ts`). When calculating derived state from these imports using O(N) operations like `.reduce()` or `.filter()`, the calculation must be moved outside the React component or memoized. Otherwise, it will trigger an expensive recalculation on every re-render (which can happen frequently with context updates like `LanguageContext`).
**Action:** When working with components that map over or group large static datasets, always check if the derived calculation is inside the functional component. If the underlying data is static (doesn't depend on component props or state), hoist the calculation to the module scope (outside the component).
